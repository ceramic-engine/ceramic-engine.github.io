ceramic.GeometryUtils (Class)

A collection of static utility functions for 2D geometric calculations.

GeometryUtils provides essential geometric operations commonly needed in 2D games and applications, including point-in-shape tests, distance calculations, angle computations, and polygon analysis.

All methods are static and optimized for performance, making them suitable for real-time applications like collision detection, input handling, and visual effects.

Common use cases:

  • Hit testing: Determine if a mouse/touch point is inside a shape
  • Collision detection: Check if circles or other shapes intersect
  • Movement: Calculate distances and angles for object positioning
  • Rendering: Determine winding order for proper triangle rendering

Example usage:

// Check if mouse is inside a triangle
if (GeometryUtils.pointInTriangle(mouseX, mouseY, v1.x, v1.y, v2.x, v2.y, v3.x, v3.y)) {
    trace("Mouse is inside triangle!");
}

// Calculate distance between two points
var dist = GeometryUtils.distance(player.x, player.y, enemy.x, enemy.y);
if (dist < 50) {
    trace("Enemy is nearby!");
}

// Get angle between player and target
var angle = GeometryUtils.angleTo(player.x, player.y, target.x, target.y);
player.rotation = angle;
See: ceramic.Visual For shape rendering, ceramic.Transform For coordinate transformations, ceramic.Triangulate For polygon triangulation

pointInTriangle(x: Float, y: Float, ax: Float, ay: Float, bx: Float, by: Float, cx: Float, cy: Float): Bool#

Tests whether a point lies inside a triangle using the sign method.

This method uses the sign of the cross product to determine if a point is on the same side of all three edges of the triangle. It handles edge cases where the point lies exactly on an edge.

Bool true if the point is inside or on the edge of the triangle, false otherwise
pointInRectangle(x: Float, y: Float, rectX: Float, rectY: Float, rectWidth: Float, rectHeight: Float): Bool#

Tests whether a point lies inside an axis-aligned rectangle.

The rectangle is defined by its top-left corner position and dimensions. Points exactly on the left or top edges are considered inside, while points on the right or bottom edges are considered outside.

intersectCircles(x0: Float, y0: Float, r0: Float, x1: Float, y1: Float, r1: Float): Bool#

Tests whether two circles intersect or touch.

Two circles intersect if the distance between their centers is less than or equal to the sum of their radii. This method uses squared distances to avoid the expensive square root operation.

distance(x1: Float, y1: Float, x2: Float, y2: Float): Float#

Calculates the Euclidean distance between two points.

This method computes the actual distance value using the square root of the sum of squared differences. For performance-critical code where you only need to compare distances, consider using squareDistance() instead.

Name Type Description
x1 Float The x-coordinate of the first point
y1 Float The y-coordinate of the first point
x2 Float The x-coordinate of the second point
y2 Float The y-coordinate of the second point
Returns Description
Float The distance between the two points
squareDistance(x1: Float, y1: Float, x2: Float, y2: Float): Float#

Calculates the squared Euclidean distance between two points.

This method returns the square of the distance, avoiding the expensive square root operation. It's ideal for comparing distances or checking if a distance is within a threshold, as the relative ordering is preserved.

Example:

// Instead of:
if (distance(x1, y1, x2, y2) < 100) { ... }

// Use:
if (squareDistance(x1, y1, x2, y2) < 100 * 100) { ... }
Name Type Description
x1 Float The x-coordinate of the first point
y1 Float The y-coordinate of the first point
x2 Float The x-coordinate of the second point
y2 Float The y-coordinate of the second point
Returns Description
Float The squared distance between the two points
angleTo(x0: Float, y0: Float, x1: Float, y1: Float): Float#

Calculates the angle from one point to another in degrees.

The angle is measured clockwise from the positive Y-axis (up), making 0° point upward, 90° point right, 180° point down, and 270° point left. This convention matches Ceramic's visual rotation system.

Name Type Description
x0 Float The x-coordinate of the starting point
y0 Float The y-coordinate of the starting point
x1 Float The x-coordinate of the target point
y1 Float The y-coordinate of the target point
Returns Description
Float The angle in degrees (0-360), measured clockwise from up
angleDelta(angle0: Float, angle1: Float): Float#

Calculates the shortest angular difference between two angles in degrees.

This method always returns the shortest path between two angles, which will be between -180 and 180 degrees. Positive values indicate clockwise rotation, negative values indicate counter-clockwise rotation.

Example:

angleDelta(350, 10) // Returns 20 (not 340)
angleDelta(10, 350) // Returns -20 (not -340)
Name Type Description
angle0 Float The starting angle in degrees
angle1 Float The target angle in degrees
Returns Description
Float The angular difference in degrees (-180 to 180)
clampDegrees(deg: Float): Float#

Normalizes an angle to be within the range [0, 360).

This method wraps angles to ensure they fall within the standard 0-360 degree range. Negative angles are wrapped to their positive equivalents, and angles greater than or equal to 360 are reduced by multiples of 360.

Examples:

clampDegrees(-90)  // Returns 270
clampDegrees(450)  // Returns 90
clampDegrees(360)  // Returns 0
Name Type Description
deg Float The angle in degrees to normalize
Returns Description
Float The normalized angle in the range [0, 360)
angleDirection(angle: Float, result: Point): Point#

Converts an angle in degrees to a unit direction vector.

The resulting vector has a magnitude of 1 and points in the direction of the given angle. The angle is measured clockwise from the positive Y-axis (up), matching Ceramic's visual rotation system.

Example:

var direction = new Point();
GeometryUtils.angleDirection(0, direction);   // direction = (0, -1) - pointing up
GeometryUtils.angleDirection(90, direction);  // direction = (1, 0)  - pointing right
GeometryUtils.angleDirection(180, direction); // direction = (0, 1)  - pointing down
Name Type Description
angle Float The angle in degrees (0° = up, 90° = right, etc.)
result Point The Point object to store the resulting direction vector
Returns Description
Point The result object for method chaining
isClockwise(vertices: Array<Float>, offset: Int, count: Int): Bool#

Determines the winding order of a polygon's vertices.

This method uses the shoelace formula to calculate the signed area of the polygon. The sign of the area indicates the winding order: negative for clockwise, positive for counter-clockwise.

Winding order is important for:

  • Determining front/back faces in rendering
  • Proper triangulation of polygons
  • Collision detection algorithms
Name Type Description
vertices Array<Float> Array of polygon vertices in the format [x0,y0,x1,y1,x2,y2,...]
offset Int Starting index in the vertices array (must be even)
count Int Number of array elements to process (must be even, minimum 6 for a triangle)
Returns Description
Bool true if vertices are ordered clockwise, false if counter-clockwise
